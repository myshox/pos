# 資料與資料庫儲存說明

## 資料存在哪裡？

目前 **所有資料都存在「瀏覽器」裡**，**不會**自動存到專案資料夾或電腦的某個檔案。

| 儲存位置 | 說明 |
|----------|------|
| **localStorage** | 商品、訂單、分類、店鋪設定、後台 PIN 都存在**瀏覽器**的 localStorage。 |
| **sessionStorage** | 後台「已解鎖」狀態（約 30 分鐘）存在 sessionStorage，關閉分頁就清除。 |

- **專案資料夾**（例如 `POS/`）裡**沒有**「資料庫」或「data」資料夾，執行時也不會自動寫入任何檔案。
- 資料是「存在你打開這個網站的那個瀏覽器」裡，例如：
  - 用 Chrome 開 → 資料在 Chrome
  - 用 iPhone Safari 開 → 資料在那支 iPhone 的 Safari
  - 換一台電腦或換一個瀏覽器 → 就是另一份資料（不會同步）。

---

## 怎麼把資料「存成檔案」、放到資料夾？

要用**備份功能**，手動把資料匯出成一個檔案，再自己存到想要的資料夾。

1. 登入後台 → **備份還原** 分頁。
2. 點 **「下載備份」**，會下載一個 **JSON 檔**（例如 `pos-backup-2025-02-03.json`）。
3. 把這個檔案存到你想要的資料夾（例如「我的文件 / POS備份」），就等於資料有存進資料夾。

之後若要還原，到 **備份還原** → **選擇檔案**，選當初存的那個 JSON 檔即可。

**建議**：重要資料請**定期**（例如每週）下載備份一次，存到固定資料夾，避免瀏覽器清除資料或換裝置時資料不見。

---

## 為什麼不自動存到資料夾？

因為這個專案是**純網頁前端**，在瀏覽器裡執行。瀏覽器**不允許**網頁隨意讀寫使用者電腦上的資料夾（安全限制），所以無法「自動寫入某個資料夾」。

若要「自動存到伺服器或檔案」，需要：
- **後端伺服器**（例如 Node.js、PHP）+ 資料庫或檔案系統，或  
- **電子桌面應用**（例如 Electron）才有權限寫入本機資料夾。

目前設計是：**瀏覽器 localStorage 當資料庫 + 手動備份到 JSON 檔**，不需架伺服器，最簡單。

## 總結

| 問題 | 答案 |
|------|------|
| 資料會存在專案資料夾嗎？ | **不會**。專案資料夾裡沒有資料庫檔案。 |
| 資料存在哪裡？ | **瀏覽器**的 localStorage（依你開網站的裝置與瀏覽器而定）。 |
| 想存到資料夾怎麼辦？ | 用後台 **備份還原 → 下載備份**，會得到一個 JSON 檔，再**手動**存到你想要的資料夾。 |
| 建議做法？ | 定期下載備份、把 JSON 存到固定資料夾，避免資料遺失。 |
